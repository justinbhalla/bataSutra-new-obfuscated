Voice & Tone

  • Clarity Prefer short sentences and concrete nouns; avoid jargon unless defined.
  • Neutrality Be fair and evidence-led; separate news from opinion.
  • Audience India-first perspectives; explain global context succinctly.

Tone sliders

  • Simple ↔ Technical: Simple unless specialist piece
  • Neutral ↔ Opinionated: Neutral
  • Formal ↔ Conversational: Conversational but precise

Grammar & Mechanics

  • Oxford comma: use it.
  • Spellings: Indian English (favour, labour), but units per SI.
  • Capitalization: sentence case for headlines; proper nouns capitalized.
  • Numerals: write out one–nine in prose; use digits from 10 onwards (context-dependent for data pieces).

Rounding

  • Rates: 2 decimals
  • Index levels: 0 decimals
  • Percent change: 1 decimal (unless tiny samples)

Formatting & Structure

  • Use clear subheads every 3–5 paragraphs; one idea per paragraph.
  • Bullets for lists; avoid walls of text.
  • Pull quotes only if the quote adds meaning (not for decoration).

Numbers, Units & Currencies

  • Label units in axis titles and first mention; INR by default. USD conversions use RBI reference or noted rates.
  • Prefer SI (k, M, B) or Indian (lakh, crore) consistently within a piece.
  • State index base (e.g., Base=2020-01) and deflators used.

Examples

₹2.3 crore; ₹125 lakh; 3.4% YoY; 12-month trailing average.

Dates, Times & Timezones

  • ISO-like dates in data contexts: 2025-09-22; prose can use “22 Sep 2025”.
  • Timezones: IST by default; note UTC offsets for global context.
  • Spell out months; avoid numeric-only formats in prose.

Links, Citations & Quotes

  • Descriptive link text (avoid “click here”).
  • Citations: “Source: RBI; SEBI; bataSutra calculations.” with direct URLs where possible.
  • Quotes are exact; ellipses/brackets only for clarity, not meaning change.

Headlines, SEO & Metadata

  • H1: one per page, descriptive and specific (≤65 chars ideal).
  • Meta description: ~150–160 chars summarising the value prop.
  • Use semantic headings (H2/H3) and meaningful slugs.

Accessibility

  • Alt text for images; captions for charts; keyboard-focusable embeds.
  • Avoid conveying meaning by color alone; provide labels/markers.
  • Respect reduced motion; no auto-playing media with sound.

Contrast

Target WCAG AA for text and UI elements.

Images, Charts & Tables

  • Charts must show units, sources, and adjustments (deflators, smoothing).
  • Zero baselines where appropriate; note trimmed axes.
  • Tables: concise headers, aligned decimals, and notes for caveats.

Code, Math & Notation

  • Inline code uses backticks; blocks are fenced and labeled with language.
  • Math: keep simple in prose; for formulas, define variables and units.
  • Version pins for reproducibility when sharing code.

Examples: Do / Don’t

Do
  • “India’s CPI rose 3.8% YoY in Aug, below RBI’s 4% target.”
  • “See chart: Base=2019-04; CPI deflated; 3-month average.”
Don’t
  • “Inflation exploded!!!” (hype, no context)
  • Unlabeled axes/units or missing sources.
